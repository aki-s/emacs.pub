概要

wsimport ツールは、次のような JAX-WS ポータブルアーティファクトを生成します。

  • サービスエンドポイントインタフェース (SEI)
  • サービス
  • wsdl:fault からマップされる例外クラス (存在する場合)
  • 応答 wsdl:message から派生する非同期応答 Bean (存在する場合)
  • JAX-B が生成する値タイプ (スキーマの型からマップされた Java クラス)

これらのアーティファクトは、WSDL ドキュメント、スキーマドキュメント、およびエンドポイント実装とともに WAR ファイル内にパッケージ化して配備することができます。また、wsimport Ant タスクも用意されています。Wsimport Ant タスクを参照してください。

wsimport の起動

  • Solaris/Linux
    • /bin/wsimport.sh -help
  • Windows
    • \bin\wsimport.bat -help

構文 

wsimport [options] <wsdl>

次の表に、wsimport のオプションを示します。

オプション

説明 

-d <directory>

生成される出力ファイルを格納する場所を指定します 

-b <path>

外部の JAX-WS または JAXB バインディングファイルを指定します (<file> ごとに -b が必要となる) 

-B <jaxbOption>

このオプションを JAXB スキーマコンパイラに渡します

 
-catalog

外部エンティティー参照を解決するためのカタログファイルを指定します。 TR9401、XCatalog、OASIS XML Catalog の各形式がサポートされています。カタログのドキュメントをお読みになり、カタログのサンプルを参照してください。 

-extension

ベンダー拡張 (仕様で規定されていない機能) を許可します。拡張を使用すると、アプリケーションの移植性が失われたり、ほかの実装との相互運用が行えなくなる可能性があります 

-help

ヘルプを表示します 

-httpproxy:<host>:<port>

HTTP プロキシサーバーを指定します (デフォルトのポートは 8080) 

-keep

生成されたファイルを保持します 

-p
このコマンド行オプション経由でターゲットパッケージを指定した場合、その指定内容は、パッケージ名に対するすべての WSDL/スキーマバインディングカスタマイズや、 仕様で規定されているデフォルトのパッケージ名アルゴリズムよりも優先されます 
-s <directory>

生成されるソースファイルを格納する場所を指定します 

-verbose

コンパイラが実行している処理に関するメッセージを出力します 

-version

バージョン情報を出力します 

-wsdllocation <location>
@WebServiceClient.wsdlLocation 値 
-target
指定された JAX-WS 仕様バージョンに従ってコードを生成します。バージョン 2.0 では、JAX-WS 2.0 仕様に準拠したコードを生成します。 
-quiet
wsimport 出力を抑制します

-b オプションを使って、複数の JAX-WS および JAXB バインディングファイルを指定できます。 これらのファイルを使って、パッケージ名や Bean 名など、さまざまなカスタマイズが可能です。 JAX-WS および JAXB バインディングファイルの詳細は、カスタマイズマニュアルを参照してください。

例 

wsimport -p stockquote http://stockquote.xyz/quote?wsdl

Java アーティファクトを生成し、http://stockquote.xyz/quote?wsdl をインポートすることで、それらのアーティファクトをコンパイルします。
